BUNDLES
Default (empty): all Vx1 VLANs → single bundle, RT asnBase:1.
Each added range → EVPN_BUNDLE_N with RT asnBase:N; device VLANs intersected per bundle.
VGW
MAC
Per-VLAN: separate EVPN instance per VLAN — RT = VLAN:VLAN.

Bundle: vlan-aware-bundle mode — RT must be identical on all VTEPs per bundle.
 • Default (no groups): single EVPN_VLAN_AWARE_BUNDLE, all Vx1 VLANs, RT = ASN_BASE:1.
 • With bundle groups: EVPN_BUNDLE_N per range, RT = ASN_BASE:N — see ⓘ under BUNDLES for a worked example.
Anycast GW: emits ip address virtual on SVIs — requires EVPN for MAC/IP Type-2 route distribution.

VARP: emits ip virtual-router address on SVIs — used with MLAG active-active gateway (EVPN optional). Requires a shared virtual-router MAC (configured below when selected).
Why bundles? In vlan-aware-bundle mode the RT must be identical on all VTEPs for routes to exchange. Two VTEPs with completely different VLAN sets need separate bundles with different RTs so each pair can still find a common RT.

Worked example — 3 VTEPs, 2 bundles (ASN_BASE = 65000):

Sheet Vx1 rows:
 • DevA  vlan_ 1-1000
 • DevB  vlan_ 2000-2999
 • DevC  vlan_ 1-2999

Bundle groups defined here:
 B1 1-1000   → RT 65000:1
 B2 2000-2999 → RT 65000:2

Generated BGP (per device, intersection):
DevA (owns 1-1000):
vlan-aware-bundle EVPN_BUNDLE_1 vlan 1-1000 route-target both 65000:1 DevB (owns 2000-2999):
vlan-aware-bundle EVPN_BUNDLE_2 vlan 2000-2999 route-target both 65000:2 DevC (owns 1-2999):
vlan-aware-bundle EVPN_BUNDLE_1 vlan 1-1000 route-target both 65000:1 vlan-aware-bundle EVPN_BUNDLE_2 vlan 2000-2999 route-target both 65000:2 Result: DevA ↔ DevC share RT 65000:1 (exchange VLAN 1-1000 routes). DevB ↔ DevC share RT 65000:2 (exchange VLAN 2000-2999 routes).

Empty list = single EVPN_VLAN_AWARE_BUNDLE with all device VLANs, RT ASN_BASE:1 (legacy / uniform VLAN sets).
